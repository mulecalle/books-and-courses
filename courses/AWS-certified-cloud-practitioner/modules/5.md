# Networking

Una Amazon VPC le permite aprovisionar una sección aislada de forma lógica de la nube de AWS, donde puede lanzar recursos de AWS en una red virtual que usted defina.

Ofrece tres beneficios principales. Ayuda a aumentar la seguridad, ya que puede proteger y supervisar las conexiones, controlar el tráfico y restringir el acceso a las instancias

VPC > Subnet (puede ser publica o privada)

Internet Gateway (public) - Una puerta de enlace de internet es una conexión entre una VPC e internet
Virtual Private Gateway (private) - Una puerta de enlace privada virtual es el componente de la nube de AWS que le permite conectar este tráfico protegido para ingresar a la VPC, requiere VPN

AWS Private connect -> magic door 1to1 with AWS

## AWS Client VPN

AWS Client VPN es un servicio de redes que puede utilizar para conectar las redes en las instalaciones y los trabajadores remotos a la nube. Es un servicio de VPN elástico y completamente administrado que se escala o reduce verticalmente de forma automática según la demanda de los usuarios

## AWS Site-to-Site VPN

Site-to-Site VPN crea una conexión segura entre el centro de datos o las sucursales y los recursos en la nube de AWS.

## AWS PrivateLink

Puede conectar de forma privada su VPC a los servicios y recursos como si estuvieran en su VPC

## AWS Direct Connect

Direct Connect es un servicio que le permite establecer una conexión privada dedicada entre su red y la VPC en la nube de AWS.

## ACL de red (stateless)

Una ACL de red es un firewall virtual que controla el tráfico entrante y saliente en el nivel de subred.
La ACL de red predeterminada de su cuenta permite todo el tráfico entrante y saliente

Las ACL de red realizan el filtrado de paquetes sin estado. No recuerdan nada y comprueban los paquetes que cruzan el límite de la subred en cada sentido: entrante y saliente.

## Security Group (stateful)

Un grupo de seguridad es el componente de VPC que revisa los permisos de los paquetes para el recurso final.
De forma predeterminada, un grupo de seguridad deniega todo el tráfico entrante y permite todo el tráfico saliente

Los grupos de seguridad realizan el filtrado de paquetes con estado. Recuerdan decisiones anteriores que se tomaron para los paquetes entrantes.

Característica | Grupos de seguridad | ACL de red 
Alcance | Nivel de instancia (conexión a instancias de EC2) | Nivel de subred (asociadas a subredes)
Estado | Con estado (recuerda el estado) | Sin estado (no recuerda el estado)
Tipos de reglas | Solo reglas del tipo “permitir” | Reglas del tipo “permitir” y “denegar”
Tráfico de retorno | El tráfico de retorno se permite automáticamente si se permite el tráfico entrante | El tráfico de retorno debe estar permitido implícitamente en ambas direcciones
Usos | Control detallado del tráfico para instancias de EC2 individuales | Amplio control del tráfico que entra y sale de las subredes

## Global Networking

### Amazon Route 53

Se puede pensar en DNS como la guía telefónica de internet. La resolución de DNS es el proceso de traducir un nombre de dominio a una dirección IP.

Route 53 dirige a los usuarios finales a sus recursos con servidores DNS dispersos por todo el mundo y escalado automático. Proporciona a los desarrolladores y los negocios una manera fiable de dirigir a los usuarios finales a las aplicaciones de internet alojadas en AWS

### Amazon CloudFront

CloudFront es un servicio de red de entrega de contenido (CDN) que entrega su contenido con tiempos de carga más rápidos, ahorros de costos y fiabilidad.

### AWS Global Accelerator

Global Accelerator es un servicio que utiliza la red global de AWS para mejorar la disponibilidad, el rendimiento y la seguridad de las aplicaciones. Utiliza un enrutamiento de tráfico inteligente y una conmutación por error rápida en caso de que haya problemas en una de las ubicaciones de las aplicaciones.
