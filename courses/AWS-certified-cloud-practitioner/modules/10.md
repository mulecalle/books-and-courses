# supervisión, el cumplimiento y la gobernanza

La progresión que normalmente se recomienda usar es la siguiente:

1 - Proteger los sistemas, los sistemas y la infraestructura contra el acceso, el uso, la divulgación, la interrupción, la modificación o la destrucción no autorizados.
2 - Supervisar las actividades, Observe y analice continuamente la actividad del sistema, el tráfico de red y los eventos de seguridad para detectar posibles amenazas o anomalías.
3 - Realizar auditorías, Revise y evalúe periódicamente la eficacia de los controles de seguridad y verifique que se cumplan todos los requisitos y las políticas y los procedimientos de seguridad.
4 - Garantizar el cumplimiento, Ayude a garantizar que las prácticas y los controles de seguridad de una organización cumplan con los requisitos de las pautas pertinentes, los estándares del sector y las obligaciones contractuales.

## Monitoring

### Amazon CloudWatch

Supervisa sus recursos de AWS y las aplicaciones que ejecuta en AWS en tiempo real. Con CloudWatch, obtiene visibilidad de todo el sistema respecto de la utilización de los recursos, el rendimiento de las aplicaciones y el estado operativo.

- Métricas de CloudWatch
- Alarmas de CloudWatch
- Paneles de CloudWatch
- Registros de CloudWatch

## Auditoria

### AWS CloudTrail

CloudTrail rastrea la actividad de los usuarios y el uso de las API en la nube de AWS, en las instalaciones e incluso con otros proveedores de la nube. CloudTrail proporciona un historial detallado de las llamadas a la API, para que pueda realizar un seguimiento de los cambios e identificar quién los realizó y cuándo.

## Compliance

El cumplimiento se refiere a que sus recursos y datos en la nube se adhieren a las normativas, los estándares del sector y las políticas internas pertinentes en relación con la seguridad y la protección de datos.

### AWS Artifact

AWS Artifact es un servicio que proporciona acceso gratuito y bajo demanda a los informes de seguridad y cumplimiento de AWS y a determinados acuerdos en línea.

Beneficios: AWS Artifact le ayuda a administrar a escala, ahorrar tiempo con el acceso bajo demanda a los informes de cumplimiento e implementar con más confianza.

AWS Artifact consta de dos tipos: Acuerdos e informes de AWS Artifact.

### AWS Config

La mayoría de las empresas querrán que siga las pautas de configuración específicas para sus diferentes recursos de AWS.

AWS Config es un servicio que puede utilizar para examinar, auditar y evaluar las configuraciones de sus recursos de AWS.

Beneficios: AWS Config ayuda a evaluar las configuraciones en función del estado deseado, administrar los cambios en la configuración de los recursos y simplificar la resolución de problemas y la corrección.

### AWS Audit Manager

Audit Manager es un servicio que audita continuamente el uso de AWS para simplificar la evaluación de riesgos y cumplimiento. Ayuda a recopilar pruebas y administrar los datos de auditoría.

### AWS Organizations

Este servicio ayuda a administrar y controlar su entorno de forma centralizada a medida que crece y escala sus recursos de AWS. Le permite administrar las políticas de los grupos de cuentas y automatizar la creación de cuentas.

Beneficios: Organizations ofrece varios beneficios, como escalar rápidamente su entorno mediante la creación programática de nuevas cuentas de AWS para los recursos y los equipos. También ayuda al simplificar la administración de permisos a través de las políticas de control de servicios (service control policies, SCP) y a administrar y optimizar los costos en sus cuentas y recursos de AWS.

Casos de uso: se puede usar para automatizar la creación de cuentas de AWS, proporcionar herramientas y acceso a sus equipos de seguridad, controlar el acceso de los usuarios a los servicios designados y compartir recursos comunes entre cuentas.

Una organización es un conjunto de cuentas de AWS que puede administrar de forma centralizada y organizar en una estructura jerárquica similar a un árbol, con una raíz en la parte superior y unidades organizativas (UO) anidadas debajo de la raíz. Cada cuenta se puede ubicar directamente en la raíz o en una de las UO de la jerarquía.

## Gobernanza

### AWS Control Tower

AWS Control Tower es un servicio que puede usar con el fin de aplicar y administrar las reglas de gobernanza para la seguridad, las operaciones y el cumplimiento a escala en todas sus organizaciones y cuentas en la nube de AWS.

Utiliza controles preconfigurados, que pueden ayudarle a configurar rápidamente entornos de cuentas múltiples, automatización con gobernanza integrada e integración de software de terceros a escala.

### AWS Service Catalog

Con Service Catalog, puede crear, compartir y organizar a partir de un catálogo seleccionado de recursos de AWS.

Beneficios: Service Catalog ahorra tiempo al facilitar la búsqueda e implementación de recursos de nube de autoservicio aprobados. 

Casos de uso: utilícelo para aprovisionar recursos en las cuentas de AWS, aplicar controles de acceso y acelerar el aprovisionamiento de canalizaciones de integración continua y entrega continua (CI/CD).

### AWS License Manager

License Manager es un servicio que le ayuda a administrar sus licencias de software y ajustar los costos relacionados.

### Panel de AWS Health

El Panel de AWS Health proporciona información valiosa como origen de datos para eventos y cambios. Le brinda orientación oportuna y práctica para solucionar los problemas. También ayuda a administrar el estado de los servicios y está integrado y automatizado para su uso a escala.

### AWS Trusted Advisor

Buscar maneras de optimizar los costos, el rendimiento, la seguridad y la resiliencia. Con AWS Trusted Advisor, puede evaluar de forma continua su entorno de AWS mediante comprobaciones de prácticas recomendadas en varias categorías.

Trusted Advisor le ayuda a alinearse con las prácticas recomendadas de AWS, priorizar las recomendaciones y optimizar sus recursos de AWS a escala.

### Analizador de acceso de IAM

El analizador de acceso de IAM ofrece funciones para establecer, verificar y refinar los permisos mediante el análisis del acceso externo y la validación de que sus políticas coincidan con los estándares de seguridad corporativos.
