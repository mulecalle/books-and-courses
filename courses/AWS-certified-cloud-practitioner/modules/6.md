# Storage

## Almacenamiento en bloque (block storage)

El almacenamiento en bloque brinda volúmenes de almacenamiento persistentes y de baja latencia a nivel de bloques que se conectan a instancias de EC2, como discos duros físicos

### Almacén de instancias Amazon EC2
Un almacenamiento en bloque no administrado, no persistente y de alto rendimiento que se conecta de forma directa a las instancias de EC2 para datos temporales.

Almacenamiento a nivel de bloques que está conectado de forma física al equipo host de la instancia de EC2. Según el tipo de instancia, el almacén de instancias EC2 puede incluirse como almacenamiento predeterminado. Dado que sus datos se pierden cuando una instancia se detiene o termina, el almacén de instancias EC2 es la mejor opción para las necesidades de almacenamiento temporal basado en memoria, como búferes, cachés y datos temporales.

Conclusión clave: no hay persistencia de datos

### Amazon Elastic Block Store (EBS)
Un servicio administrado que brinda volúmenes de almacenamiento en bloque persistentes para instancias de EC2 y ofrece varios tipos para cargas de trabajo diferentes.

Amazon EBS proporciona volúmenes de almacenamiento persistente a nivel de bloques para utilizarlos con instancias de Amazon EC2. Los volúmenes de EBS actúan como discos duros externos y ofrecen un rendimiento uniforme y de baja latencia para cargas de trabajo, como bases de datos y sistemas de archivos.

Conclusión clave: persistencia de datos

Rollback: Las instantáneas de EBS son copias de seguridad de un volumen de EBS en un momento dado. Se pueden utilizar para la recuperación de desastres, la migración de datos, el redimensionamiento de volúmenes y para crear copias de seguridad consistentes de las cargas de trabajo de producción. Estas instantáneas son progresivas, por lo que solo guardan los bloques del volumen que se modificaron después de la última instantánea.

## Almacenamiento de objetos (object storage)

El almacenamiento de objetos es una arquitectura de almacenamiento de datos que administra los datos como objetos en un espacio de direcciones plano. Ofrece escalabilidad ilimitada para que almacene grandes cantidades de datos no estructurados sin preocuparse por las restricciones de capacidad. El almacenamiento de objetos proporciona capacidades de metadatos mejoradas de modo que la administración, la búsqueda y el análisis de datos sean más eficientes en conjuntos de datos masivos

### Amazon Simple Storage Service (S3)
Un servicio de almacenamiento de objetos escalable y completamente administrado para almacenar y recuperar cualquier cantidad de datos desde cualquier lugar.

Ofrece una durabilidad del 99,999999999 %, lo que significa que sus datos cuentan con una alta protección contra pérdidas

Bucket: Los buckets tienen un nombre único a nivel global en todo AWS, lo que ayuda a identificar y organizar los datos almacenados

Seguridad:

Las políticas de buckets de Amazon S3 son políticas basadas en recursos que solo se pueden conectar a los buckets de S3. Una política de bucket de S3 especifica qué acciones se permiten o no en el bucket, además de cada objeto de ese bucket.

Los permisos que regulan las acciones que los usuarios, los grupos o los roles pueden realizar en los recursos de S3 se configuran con políticas basadas en identidad. Estas políticas se vinculan de forma directa a las identidades en lugar de a los propios recursos de S3. Con estas políticas, puede especificar a qué buckets y objetos de S3 pueden acceder los usuarios y qué acciones pueden realizar.

Cifrado: 
El cifrado en reposo protege los datos almacenados en buckets de S3, lo que evita el acceso no autorizado a los objetos almacenados.
El cifrado en tránsito protege los datos que se trasladan hacia y desde Amazon S3 y mantiene una comunicación segura entre los clientes y el servicio.

S3 Intelligent-Tiering:
Almacena los objetos en tres niveles: un nivel de acceso frecuente, un nivel de acceso poco frecuente y un nivel de acceso instantáneo al archivo. Amazon S3 supervisa los patrones de acceso de sus datos y los traslada de forma automática al nivel de almacenamiento más rentable en función de la frecuencia de acceso

S3 Standard - Acceso poco frecuente (Standard-IA)
Se utiliza para los datos a los que se accede con menor frecuencia, pero que requieren un acceso rápido cuando es necesario. S3 Standard-IA ofrece la alta durabilidad, el alto rendimiento y la baja latencia de S3 Standard, con precios económicos de almacenamiento y recuperación por GB. Este nivel de almacenamiento es ideal si desea almacenar copias de seguridad a largo plazo, archivos de recuperación de desastres

S3 One Zone - Acceso poco frecuente (One Zone-IA)
Almacena los datos en una única zona de disponibilidad, lo que reduce los costos en comparación con S3 Standard-IA, que utiliza tres zonas

S3 Express One Zone
Almacena los datos en una única zona de disponibilidad. Se diseñó con el fin de brindar acceso constante en milisegundos de un solo dígito para los datos de acceso frecuente y las aplicaciones sensibles a la latencia

S3 Glacier Instant Retrieval
archivar datos de acceso muy poco frecuente y que requieren una recuperación en milisegundos. Los datos que se almacenan en esta clase de almacenamiento implican un ahorro de costos de hasta un 68 por ciento en comparación con la clase de almacenamiento S3 Standard-IA, con la misma latencia y rendimiento

S3 Glacier Flexible Retrieval
almacenamiento de bajo costo para los datos archivados a los que se accede una o dos veces al año

S3 Glacier Deep Archive
Los datos que se almacenan en la clase de almacenamiento S3 Glacier Deep Archive tienen un tiempo de recuperación predeterminado de 12 horas. Está diseñado para clientes que retienen conjuntos de datos por un período de 7 a 10 años o más, a fin de cumplir con los requisitos de conformidad normativa

S3 Outposts
Amazon S3 Outposts ofrece almacenamiento de objetos a su entorno en las instalaciones de AWS Outposts mediante las características y la API de Amazon S3, y está pensado para cargas de trabajo con requisitos de residencia local de datos

## Almacenamiento de archivos (file storage)

Los servicios de almacenamiento de archivos de AWS brindan sistemas de archivos compartidos a los que se puede acceder a través de redes, de modo que varios usuarios y aplicaciones pueden acceder a los mismos datos de forma simultánea

### Amazon Elastic File System (EFS)
Un sistema de archivos NFS escalable y completamente administrado para utilizar con los servicios en la nube de AWS y los recursos en las instalaciones.

Funciona mediante el protocolo del sistema de archivos de red (NFS) en Linux y se escala de forma automática a petabytes a medida que se agregan o eliminan archivos sin interrumpir las aplicaciones

Las clases de almacenamiento EFS Standard y EFS Standard - Acceso poco frecuente (Standard-IA) ofrecen resiliencia Multi-AZ y los niveles más altos de durabilidad y disponibilidad. Tienen un costo más alto debido a su mayor disponibilidad y durabilidad.

Las clases EFS One Zone y EFS One Zone - Acceso poco frecuente (EFS One Zone-IA) proporcionan ahorros adicionales, ya que guardan los datos en una única zona de disponibilidad. Si utiliza solo una zona de disponibilidad, puede reducir los costos de almacenamiento en comparación con las clases de almacenamiento EFS Standard.

La clase de almacenamiento EFS Archive permite reducir los costos para los datos a los que se accede solo unas pocas veces al año o menos y que no requieren las latencias inferiores a milisegundos de EFS Standard

### Amazon FSx
Amazon FSx admite varios protocolos de sistemas de archivos, incluidos:

Windows File Server, brinda almacenamiento compartido completamente administrado creado a partir de Windows Server

Lustre, proporciona almacenamiento compartido completamente administrado con la escalabilidad y el rendimiento del popular sistema de archivos Lustre

OpenZFS, creado a partir del sistema de archivos OpenZFS y al que se accede a través del protocolo NFS (v3, v4, v4.1 y v4.2)

NetApp ONTAP, proporciona almacenamiento compartido completamente administrado en la nube de AWS con las populares capacidades de acceso y administración de datos de ONTAP

## Servicios de almacenamiento adicionales

### AWS Storage Gateway
Un servicio de almacenamiento en la nube híbrida completamente administrado que proporciona acceso en las instalaciones a almacenamiento en la nube casi ilimitado.

Optimizar la administración del almacenamiento y reducir los costos en casos de uso prácticos de almacenamiento en la nube híbrida

Amazon S3 File Gateway: Amazon S3 File Gateway une su entorno local con Amazon S3. Brinda a las aplicaciones en las instalaciones acceso a un almacenamiento en la nube casi ilimitado a través de protocolos de archivos conocidos. S3 File Gateway permite almacenar y recuperar objetos en la nube mediante operaciones de archivos conocidas.

Volume Gateway: Funciona como un puente entre la infraestructura en las instalaciones y el almacenamiento en la nube de AWS, ya que presenta los datos en la nube como volúmenes iSCSI que las aplicaciones existentes pueden montar.

Tape Gateway: Tape Gateway brinda una interfaz que funciona con el software existente de copias de seguridad en cinta, por lo que la transición de las cintas físicas al almacenamiento en la nube es fluida

### Recuperación elástica ante desastres de AWS
Un servicio completamente administrado que optimiza la recuperación de sus servidores físicos, virtuales y basados en la nube en AWS.

## Amazon Data Lifecycle Manager

Puede automatizar la creación, la retención y la eliminación de instantáneas de EBS con Amazon Data Lifecycle Manager. Este servicio puede programar instantáneas fuera de las horas de uso máximo para minimizar el impacto en el rendimiento y eliminar de forma automática las copias de seguridad obsoletas, a fin de controlar los costos de almacenamiento. En particular, es valioso para las implementaciones a gran escala en las que la administración manual de instantáneas llevaría mucho tiempo y sería propensa a errores.
