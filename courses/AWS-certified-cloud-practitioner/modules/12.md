# Migración a la nube de AWS

Las tres fases del proceso de migración

- Evaluar:

AWS Migration Evaluator: es un servicio de evaluación de la migración que le ayuda a crear un caso empresarial para la planificación y la migración a la nube de AWS. Lo hace con un enfoque basado en datos: analiza su estado actual y su estado objetivo y, luego, desarrolla un plan de preparación para la migración con los costos proyectados de la nube.

- Movilizar:

Application Discovery Service: descubre el inventario y las conexiones de los servidores en las instalaciones. Reúne los detalles de configuración, rendimiento y conexión de los servidores y las bases de datos para crear un plan de migración detallado.
Migration Hub: es un centro unificado que le ofrece una guía durante las fases de descubrimiento, evaluación, planificación y ejecución de la migración. Proporciona herramientas, orientación y recomendaciones automatizadas para colaborar con su equipo y realizar un seguimiento de la migración.

- Migrar y Modernizar:

Application Migration Service: es una herramienta que sirve para mover y mejorar sus aplicaciones en las instalaciones y basadas en la nube. Ayuda a los clientes a optimizar y acelerar la migración y modernización de las aplicaciones, además de reducir sus costos.

## AWS Cloud Adoption Framework (AWS CAF)

AWS CAF es un marco de trabajo que aporta la experiencia y las prácticas recomendadas de AWS a las empresas que se preparan para migrar a la nube de AWS. El marco de trabajo proporciona herramientas para ayudar a acelerar el proceso de migración, organizar los recursos y alinear la administración durante la transición.

## 7 estrategias de migración

### Reubicar
Reubicarse es cambiar la ubicación del alojamiento a la nube. Esto podría ocurrir si las aplicaciones ya son máquinas virtuales (VM) o contenedores que se ejecutan en las instalaciones y luego se trasladan a la nube.

### Realojar
Volver a alojar, también conocida como levantar y mover, implica mover las aplicaciones sin cambios.

En un escenario de migración de sistemas heredados de gran envergadura, en el que la empresa busca implementar su migración y escalar rápidamente para cumplir con un caso empresarial, la mayoría de las aplicaciones se vuelven a alojar.

### Redefinir la plataforma
Redefinir la plataforma, también conocida como levantar, remendar y mover, implica llevar a cabo algunas optimizaciones en la nube para lograr un beneficio tangible. La optimización se logra sin cambiar la arquitectura central de la aplicación.

### Refactorizar
Refactorizar, también conocida como reestructurar, implica volver a pensar la forma en que se diseña y desarrolla una aplicación mediante el uso de características creadas para la nube. La refactorización está impulsada por una fuerte necesidad empresarial de agregar características, escalar o mejorar el rendimiento que, de otro modo, serían difíciles de lograr en el entorno actual de la aplicación.

### Recomprar
Implica pasar de una licencia tradicional a un modelo de software como servicio (SaaS).

Por ejemplo, una empresa puede optar por implementar la estrategia de recompra migrando de un sistema de administración de relaciones con los clientes (customer relationship management, CRM) a un nuevo software de fuerza de ventas.

### Retener
La retención consiste en mantener las aplicaciones críticas para el negocio en el entorno de origen. Esto podría incluir aplicaciones que requieran una gran refactorización a fin de poder migrarse o trabajos que pueden posponerse para más adelante.

### Retirar
El retiro es el proceso de eliminar aplicaciones que ya no son necesarias.

## DB Migration

### AWS DMS

AWS Database Migration Service (AWS DMS) permite migrar bases de datos de forma rápida y segura y realizar tareas continuas de replicación de datos para bases de datos y almacenes de datos activos. Proporciona una forma de planificar, evaluar, convertir y migrar bases de datos, incluso con almacenes de datos, en una herramienta central.

### AWS SCT

La migración de la base de datos a la nube puede brindar la oportunidad de rediseñar y mejorar la arquitectura de la base de datos. Dado que los requisitos y los sistemas heredados cambian con el tiempo, también puede considerar la posibilidad de migrar a un servicio de base de datos administrada por AWS o a una base de datos de código abierto para reducir los costos de licencias.

Si desea cambiar de una base de datos comercial a una de código abierto durante la migración, hay algunos aspectos logísticos importantes. Cuando las bases de datos que está migrando tienen diferentes motores de origen y destino, debe considerar cómo recrear los recursos existentes, como el esquema de la base de datos, en el destino.

AWS SCT permite convertir esquemas de bases de datos y objetos de código (como procedimientos, vistas y funciones almacenados) de un motor de base de datos a otro. AWS SCT puede incluso proporcionarle estimaciones del esfuerzo que supone una conversión, lo que ayuda a planificar.

## Transferencia de datos en línea

Varios servicios de AWS facilitan la transferencia de datos en línea a la nube de AWS. En la última lección, aprendió que AWS Database Migration Service (AWS DMS) transfiere la base de datos y sus datos a la nube de AWS. En esta lección, identificará los servicios que pueden ayudar con las consideraciones de la transferencia en línea de otros tipos de datos y archivos.

### AWS DataSync

AWS DataSync está diseñado específicamente para automatizar y acelerar la transferencia de datos. DataSync simplifica y acelera la transferencia de grandes cantidades de datos entre el almacenamiento en las instalaciones y los servicios de almacenamiento de AWS, como Amazon Simple Storage Service (Amazon S3). Automatiza muchos aspectos del proceso de transferencia, que incluyen las instancias en ejecución, el cifrado y la optimización de la red.

### AWS Transfer Family

AWS Transfer Family permite administrar y compartir datos sin interrupciones mediante transferencias de archivos sencillas, seguras y escalables. Este servicio proporciona soporte completamente administrado para transferencias seguras de archivos a través del FTP, el Protocolo de transferencia segura de archivos (SFTP), el Protocolo seguro de transferencia de archivos (FTPS) y otros protocolos. Le ayuda a transferir archivos directamente hacia y desde los servicios de almacenamiento de AWS, como Amazon S3 y Amazon EFS.

### Direct Connect

AWS Direct Connect es un servicio que le permite establecer una conexión privada dedicada entre su red y la nube privada virtual (VPC) en la nube de AWS. Como es su conexión dedicada, es una forma rápida, fiable y segura de transferir sus datos o archivos.

## Transferencia de datos sin conexión

Dispositivos de Snowball Edge Storage Optimized

Los dispositivos de AWS Snowball Edge Storage Optimized son una excelente solución para la migración de datos sin conexión cuando la conexión a internet puede no ser una opción. Estos dispositivos ofrecen almacenamiento NVMe de alto rendimiento, lo que permite simplificar las migraciones de datos de varios petabytes desde ubicaciones en las instalaciones a AWS.
